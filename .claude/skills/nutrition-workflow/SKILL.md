# 营养方案完整工作流

一站式完成个性化营养方案全流程。

## 输入

`$ARGUMENTS`：文件路径（开始新流程）| `resume`（继续中断流程）| 留空（手动输入信息）

## 执行流程

### Phase 1: 信息收集

**有文件** → Read 读取（PDF/图片），提取：基本信息、疾病诊断、异常指标、用药、过敏
**无文件** → 分轮次引导用户输入：
1. 性别/年龄/身高/体重/腰围/慢病/过敏
2. 检查指标（血糖/血脂/肝肾功能）和用药
3. 活动水平/体重目标/饮食偏好/烹饪条件

### Phase 2: 健康档案生成

生成结构化健康档案，包含：基本信息、健康状况（诊断/异常/过敏/用药）、营养风险评估

保存到 `results/<姓名>/健康档案.md`，**向用户展示并确认后继续**。

### Phase 3: 检索营养方案参考

根据诊断构造 2-3 个 WebSearch 检索词（权威营养指南/疾病膳食方案），读取有价值页面，保存到 `knowledge/references/参考_营养评估_[关键词]_YYYY-MM-DD.md` 并验证。**告知用户检索来源概要后继续**。

### Phase 4: 营养评估

读取 `disease-diet-rules.md`、`nutrient-reference.md` 和 Phase 3 的参考文件，计算营养目标：
1. BMR (Mifflin-St Jeor) → TDEE (×活动系数) → 体重目标调整
2. 疾病调整（逐病+多病叠加取最严）
3. 药物-营养交互检查
4. 可用 `scripts/calculate-nutrition.js` 验证计算结果

生成完整报告（能量/宏量营养素/微量营养素/饮食原则/推荐禁忌食物/三餐分配/参考来源），保存到 `results/<姓名>/营养评估.md`。**展示摘要确认后继续**。

### Phase 4.5: 运动处方生成

基于健康档案和营养评估，检索运动指南参考（2-3 WebSearch），保存并验证。
生成运动处方：安全性评估 → 处方参数 → 运动明细 → 一周计划 → 运动与营养协同 → 注意事项 → 循序渐进计划
强度/频率等参数可用 `scripts/exercise-data.js` 查表。

保存到 `results/<姓名>/运动处方.md`。**展示摘要确认后继续**。

### Phase 5: 检索菜谱参考

构造 3-5 个检索词（配餐方案+菜品做法+食材替换），WebSearch 并保存到 `knowledge/references/`。**告知用户检索来源后继续**。

### Phase 6: 配餐生成

询问用户天数（1/7/30天），按营养目标生成配餐方案。
- 营养达标（偏差>10%修正）、禁忌遵守、食物多样化（≥12种/天）
- 菜品不重复（一周）、核算校验（±1g/50kcal/±5%）
- 可用 `scripts/calculate-nutrition.js` 的 `validateMealTotals`/`checkEnergyConsistency` 验证

保存配餐方案（一月方案拆分为 `配餐方案_第1~4周.md`）。**展示并询问调整**。

### Phase 7: 数据转换

执行 `/data-transform <姓名>` 生成结构化数据。

## Resume 功能

根据已有文件的完整度判断中断位置：
- 有健康档案无营养评估 → 从 Phase 3 继续
- 有营养评估无运动处方 → 从 Phase 4.5 继续
- 有运动处方无配餐 → 从 Phase 5 继续
- 全部都有 → 展示摘要，询问调整内容

## 交互风格

- 分阶段暂停确认，不一次输出所有内容
- 关键数字用粗体标注
- 每阶段结尾说明下一步计划
- 每个输出文件末尾必须包含免责声明
