# 配餐生成 Skill

基于营养评估报告，检索公开菜谱参考，生成科学可执行的配餐方案。

## 输入

`$ARGUMENTS`：`<天数> [周数N] [评估文件路径]`，示例: `/meal-plan 7 2`（第2周）

## 处理步骤

### 步骤 1: 读取营养评估报告

提取：每日营养目标、三餐能量分配、推荐/禁忌食物清单、饮食原则、过敏原

### 步骤 2: 读取参考资料

读取 `knowledge/food-nutrition-guide.md`、`knowledge/meal-templates.md`、`knowledge/disease-diet-rules.md` 和 `knowledge/references/` 下的已有参考文件。

### 步骤 3: 检索公开菜谱参考（必须执行）

至少 **3-5 个 WebSearch**（同类疾病配餐方案 + 具体菜品做法 + 食材替换），保存到 `knowledge/references/参考_配餐_[关键词]_YYYY-MM-DD.md` 并验证。

优先来源：下厨房/豆果美食、丁香医生/好大夫、医院营养科食谱、中国营养学会膳食示例

### 步骤 4: 询问用户偏好（如首次）

如 `results/<用户名>/用户偏好.md` 不存在，询问：饮食偏好(中式/西式/混合)、口味(清淡/一般/重口)、烹饪条件、预算。保存到该文件，后续跳过。

### 步骤 5: 生成配餐方案

#### 核心原则
1. **营养达标** — 每餐和每日必须满足目标，偏差>±10%自动修正
2. **禁忌严格遵守** — 禁忌食物不出现
3. **食物多样化** — 每天≥12种食物，一周≥25种
4. **实操性强** — 食材常见、烹饪合理

#### 菜品多样性规则
- 同一蛋白质来源不连续2天同餐次
- 同种蔬菜一周≤3次，不连续2天当主菜
- 一周≥4种不同主食
- 同种食材至少2种做法
- 一周蛋白质来源≥4种(蛋/鱼/禽/畜/豆制品)

#### 营养核算
- 每餐展示逐食材明细表（食材/重量/能量/蛋白质/脂肪/碳水/纤维）
- **三餐加总校验**: 早餐+午餐+晚餐的各营养素加总与日目标偏差≤1g
- **能量守恒校验**: 蛋白质×4 + 碳水×4 + 脂肪×9 与标注总能量偏差≤50kcal
- **周均校验**: 7天日均值与目标偏差≤5%
- 用脚本验证：
```bash
node -e "
const { validateMealTotals, checkEnergyConsistency } = require('./scripts/calculate-nutrition');
// 示例: 检查实际值是否在目标±10%内
console.log(validateMealTotals({protein:89, carbs:215, fat:45, fiber:22}, {protein:90, carbs:215, fat:45, fiber:23}));
// 检查能量一致性
console.log(checkEnergyConsistency({protein:89, carbs:215, fat:45}, 1620));
"
```

#### 输出格式
每日配餐包含：早餐/午餐/晚餐/加餐（食物表+营养明细+今日营养汇总表与目标对比+校验状态）
一周/一月方案还需：周期营养总览表（日均值对比）、采购清单（分耐储存/生鲜）、体重管理追踪提示

### 步骤 6: 生成采购清单

按周期和食材类别汇总，一周方案分"耐储存"和"每日采购"，一月方案按周分解。

### 步骤 7: 保存方案

保存到 `results/<用户名>/配餐方案_第N周.md`（一月方案按周拆分多个文件）。
支持多周续写：已有第1周时后续生成第2周、第3周等。

## 完成后操作

自动执行 `/data-transform <用户名>`。

## 注意事项

- 份量用家用度量（碗/个/片/两），烹饪优先蒸煮炖凉拌，用油每餐5-10g
- 检索到的菜谱只作参考和灵感，最终方案须通过营养核算验证
- 每个输出文件末尾必须包含免责声明
