# 对话式信息录入 Skill

通过多轮对话引导用户提供健康信息，生成与 `parse-medical-record` 相同格式的健康档案。

适用于无病历文件、用户直接口述信息的场景。

## 输入

- `$ARGUMENTS` 留空：新用户，从零开始对话采集
- `$ARGUMENTS` 指定姓名：补录信息，加载已有 `/tmp/<姓名>/健康档案.md` 继续

## 处理步骤

### Round 1: 基础信息

每次只问 1 个问题，等待用户回答后再问下一个。按以下顺序：

1. 姓名 → 脱敏为 `X先生/女士`
2. 性别
3. 年龄
4. 身高 (cm)
5. 体重 (kg) → 收到后自动计算 BMI
6. 腰围 (cm，可选)
7. 慢性病史（高血压/糖尿病/高血脂/痛风/CKD/其他）
8. 过敏史（食物/药物）

全部问完后汇总确认，再进入 Round 2。

### Round 2: 检查指标与用药

先问是否有近期的检查报告。如有则逐项录入，每次问 1 项：

- 血糖血脂：空腹血糖、HbA1c、总胆固醇、LDL-C、HDL-C、甘油三酯
- 肝肾功能：ALT、AST、肌酐、BUN、尿酸
- 血压（如有）

用药情况：当前用药（药名、剂量、用途），关注药物-营养交互（二甲双胍→B12、他汀→避免西柚、利尿剂→补钾）

用户不知道具体数值时标记"未提供"，不要追问。全部问完后确认进入 Round 3。

### Round 3: 生活方式与目标

每次问 1 个问题：
- 活动水平：久坐 / 轻度活动 / 中度活动 / 高度活动
- 体重目标：减重 / 维持 / 增重
- 饮食偏好：中式 / 西式 / 混合
- 口味：清淡 / 一般 / 重口
- 烹饪条件（如厨房设备限制）
- 吸烟 / 饮酒习惯

全部问完后确认进入下一步。

### 步骤 4: 生成健康档案

计算 BMI，按以下格式生成健康档案，与 parse-markdown.js 兼容：

```markdown
# 健康档案

## 基本信息
- 姓名: X先生/女士  性别:  年龄:  身高: cm  体重: kg
- BMI: [数值]（偏瘦<18.5 / 正常18.5-24 / 超重24-28 / 肥胖≥28）
- 腰围: cm（如有）
- 过敏: [无/列表]

## 检验指标

### 血液检查
| 指标 | 结果 | 单位 | 参考范围 | 判定 |
|------|------|------|---------|------|
| （按实际信息填写，无则省略此行）| | | | |

### 其他检查

## 诊断
1. [慢性病列表，无则填"无明确诊断"]

## 用药
| 药物 | 剂量 | 用途 | 备注 |
|------|------|------|------|
| （按实际用药填写）| | | |

## 生活方式
- 吸烟/饮酒/运动习惯/活动水平

## 营养重点关注方向
[基于收集到的信息总结2-3个营养干预方向]
```

保存到 `/tmp/<姓名>/健康档案.md`（创建 /tmp/<姓名>/ 目录）。

### 步骤 5: 完成

向用户展示健康档案摘要，提示下一步可运行 `/nutrition-assess <姓名>`。

## 注意事项

- 每次只问 1 个问题，等待用户回答后再继续
- 用户不知道或不想回答的项标记"未提供"，不要强求
- BMI 自动计算，计算公式：体重kg / (身高m)²
- 判定栏用：正常 / 偏高 / 偏低 / 临界
- **免责声明**：本记录仅用于营养方案参考，不构成医疗诊断
