# Meal Plan

基于用户的营养评估报告，生成科学、美味、可执行的配餐方案。

## 输入

- `results/<姓名>/营养评估.md`
- `results/<姓名>/用户偏好.md`（如存在）

## 处理步骤

### 步骤 1: 读取营养评估
提取：每日营养目标、三餐能量分配、推荐/禁忌食物、饮食原则、过敏原。

### 步骤 2: 读取本地知识库
- `knowledge/food-nutrition-guide.md`
- `knowledge/meal-templates.md`
- `knowledge/disease-diet-rules.md`
- `knowledge/references/` 下已有的参考文件

### 步骤 3: 检索公开菜谱和配餐方案（强制）

必须执行至少 **3 个 WebSearch 查询**并保存到 `knowledge/references/`。

检索词示例：
- `"糖尿病 一周食谱 1500千卡 低GI"`
- `"高血压 DASH饮食 一周食谱"`
- `"荞麦面做法 糖尿病友好"`
- `"1500千卡减脂食谱 一周不重复"`

优先来源：下厨房、豆果美食、丁香医生、好大夫、医院营养科食谱、中国营养学会膳食示例。

保存格式：
- 文件名：`参考_配餐_[关键词]_YYYY-MM-DD.md`
- 内容：核心菜品/方案、营养数据、适用性评估

保存后使用 Read 工具验证头部 5 行。

### 步骤 4: 询问用户偏好（首次）

如果 `results/<姓名>/用户偏好.md` 不存在，简要询问：
1. 饮食偏好：中式/西式/混合？
2. 口味偏好：清淡/一般/重口味？
3. 烹饪条件：有时间做三餐 / 只能做一两餐 / 外卖为主？
4. 预算：日常 / 较宽裕？
5. 特别要求：宗教饮食、素食等

保存到 `results/<姓名>/用户偏好.md`。

### 步骤 5: 生成配餐方案

#### 核心原则
1. 营养达标（偏差>10%自动修正）
2. 禁忌食物绝对不出现
3. 食物多样化（每天≥12种，一周≥25种）
4. 实操性强，食材常见
5. 照顾用户口味偏好
6. 优先融合检索到的公开菜品

#### 菜品多样性规则（一周/一月方案必须遵守）
- 同种蛋白质不连续2天出现在同一餐次
- 同种蔬菜一周内不超过3次，且不连续2天作为主菜
- 一周内至少使用4种不同主食
- 同种食材至少用2种不同做法
- 蛋白质来源覆盖≥4种（鸡蛋+鱼+禽肉+畜肉+豆制品）

#### 每餐结构
- **早餐** (~30%)：主食1份 + 蛋白质1份 + 蔬果1份
- **午餐** (~40%)：主食1份 + 蛋白质1-2份 + 蔬菜2份 + 可选汤
- **晚餐** (~30%)：主食0.5-1份 + 蛋白质1份 + 蔬菜2份
- **加餐** (可选)：水果/坚果/酸奶，150-200 kcal

#### 营养核算
- **一日方案**：每餐必须包含逐食材明细计算表
- **一周/一月方案**：提供每日营养汇总表，任选一天展示完整明细作为示范

#### 达标校验（必须）
- 三餐加总与每日目标对比，蛋白质/碳水偏差>1g或脂肪>1g必须调整
- 能量守恒：蛋白质×4 + 碳水×4 + 脂肪×9 与总能量偏差>50kcal需修正
- 周均与目标吻合，偏差>5%需调整
- 在每日营养汇总表中标注校验状态（✓ 通过 / ⚠ 已修正）

#### 按周期不同要求
- **1天**：详细做法步骤 + 每餐完整核算
- **7天**：周一到周日不重复 + 采购清单 + 一周总览
- **30天**：按4周规划，每周主题变换，附月度采购清单和阶段性提示

### 步骤 6: 输出

保存到 `results/<姓名>/配餐方案_第1周.md`（或第N周）。

文件末尾包含：
- 参考来源
- 免责声明

## 输出后

展示配餐方案摘要，询问是否需要调整食材/份量/某天方案，或继续 `/data-transform`。
